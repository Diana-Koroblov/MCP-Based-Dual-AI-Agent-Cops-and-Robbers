### Part A: Google Cloud Console Configuration (Creating the Client)

1. **Create a New Project:**
* Go to the [Google Cloud Console](https://console.cloud.google.com/) and ensure you are logged into your free, personal Gmail account.
* Click the **Select a project** dropdown at the top left, then click **New Project**.
* Name your project using lowercase letters and underscores only (**snake_case** format, e.g., `group_name_007`). Click **Create**.
* Make sure your new project is actively selected in the top dropdown menu.


2. **Enable the APIs (Gmail & Calendar):**
* In the top search bar, type **Gmail API**, click on the result, and click **Enable**.
* To enable the Calendar API: Click the three-line menu icon (hamburger menu) on the top left -> go to **APIs & Services** -> **Library**.
* Search for **Google Calendar API**, select it, and click **Enable**.


3. **Configure the OAuth Consent Screen:**
* In the left sidebar, navigate to **OAuth consent screen** (or click **Get Started** if prompted).
* Select **External** as the User Type (this allows you to test with a free personal account) and click **Create**.
* Fill in the required fields: App name (e.g., `app_for_python_user`), your email for **User support email**, and your email under **Developer contact information**. Click **Save and Continue**.


4. **Set Scopes (Data Access):**
* In the **Data Access** / **Scopes** section, click **Add or remove scopes**.
* Manually add the specific scope URLs for Gmail (to read/write drafts) and Google Calendar (to manage events). Click **Update** to add them to your table, then save.


5. **Create the Desktop Client Credentials (JSON):**
* Go to the **Credentials** tab in the left sidebar.
* Click **+ Create Credentials** at the top and select **OAuth client ID** (the second option).
* For **Application type**, select **Desktop App**.
* Give it a recognizable name (e.g., `desktop_client_for_gmail`) and click **Create**.
* A pop-up will appear. Click the **Download JSON** button to save the credential file to your computer.
* > **Lecturer's Security Note:** Move this downloaded JSON file into a private, dedicated folder **outside** of your Python project directory (e.g., a folder named `private_google`). Rename the file to something short and simple, like `my_google.json`.




6. **Add Test Users (Crucial Step):**
* Go back to the **OAuth consent screen** tab, scroll down to the **Test users** section, and click **Add users**.
* Enter your exact personal Gmail address (and your project partners' emails, if applicable). Without this step, Google will block your Python script from authenticating. Click **Save**.
